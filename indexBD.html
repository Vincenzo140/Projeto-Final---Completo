Modelagem de Dados

DER

DER (Diagrama Entidade-Relacionamento): É uma representação gráfica que descreve as entidades (objetos ou conceitos) envolvidos em um sistema e os relacionamentos entre eles. O DER é composto por entidades, atributos e relacionamentos. As entidades são os objetos principais que queremos armazenar no banco de dados, como clientes, produtos ou pedidos. Os atributos são as características ou propriedades dessas entidades, como nome, idade ou preço. Os relacionamentos mostram como as entidades estão conectadas, como um cliente faz um pedido ou um produto pertence a uma determinada categoria. O DER nos ajuda a visualizar a estrutura do banco de dados e a compreender as relações entre as entidades.

MER

MER (Modelo Entidade-Relacionamento): É uma representação conceitual e abstrata dos dados que serão armazenados em um banco de dados. O MER é uma etapa anterior ao DER e oferece uma visão mais ampla e independente de qualquer sistema específico. Ele descreve as entidades principais, seus atributos e os relacionamentos entre elas. O objetivo do MER é fornecer uma representação clara e precisa da estrutura de dados, independentemente da implementação técnica. Ele permite que os projetistas de banco de dados compreendam os requisitos do sistema e planejem a estrutura de armazenamento dos dados de forma eficiente.

Onde posso criar o DER e MER?

Bom para você criar o DER e o MER, você pode utilziar ferramentas online's como o Draw.io. Outras opções populares incluem o Lucidchart, o Gliffy e o Creately. Essas ferramentas também oferecem recursos semelhantes, permitindo que você crie DERs e MERs de maneira visual e mais interativa.

Linguagem SQL

DQL

DQL (Data Query Language) é uma parte do SQL (Structured Query Language) que se concentra na recuperação de dados de um banco de dados. Ele é usado para executar consultas e extrair informações específicas das tabelas. Foram usados alguns comando DQL nesse trabalho, como:

- SELECT: comando mais comumente usado em DQL. Ele recupera dados de uma ou mais tabelas com base em critérios específicos.

- FROM: Especifica a tabela ou tabelas das quais os dados serão recuperados.

- ORDER BY: Classifica os resultados em ordem ascendente ou descendente com base em uma ou mais colunas.

- GROUP BY: Agrupa os resultados com base em uma ou mais colunas e permite a aplicação de funções de agregação, como COUNT, SUM, AVG, etc.

O comando WHERE desempenha um papel crucial no DML ao realizar operações de consulta, atualização ou exclusão de dados. O WHERE é usado para especificar condições que filtram os registros que serão afetados pela operação.

DML

A DML (Data Definition Language) lida com a definição das estruturas de dados no banco de dados.

- SELECT: Recupera registros de uma tabela ou tabelas relacionadas.

- UPDATE: Atualiza registros existentes em uma tabela.

- DELETE: Exclui registros de uma tabela.

O comando WHERE desempenha um papel crucial no DML ao realizar operações de consulta, atualização ou exclusão de dados. O WHERE é usado para especificar condições que filtram os registros que serão afetados pela operação.

Se você não usar a cláusula WHERE no DML, as operações de consulta, atualização ou exclusão serão aplicadas a todos os registros da tabela-alvo. Isso pode resultar em efeitos indesejados, como a modificação ou exclusão de todos os registros em uma tabela, ou a recuperação de todos os registros em uma consulta, incluindo aqueles que você pode não precisar.

DDL

A DDL (Data Definition Language) lida com a definição das estruturas de dados no banco de dados.

- CREATE TABLE: Cria uma nova tabela no banco de dados.

- ALTER TABLE: Modifica a estrutura de uma tabela existente, como adicionar, modificar ou excluir colunas.

- DROP TABLE: Exclui uma tabela do banco de dados.

Categoria de comandos permite a definição das estruturas que organizam os dados e garantem a integridade do banco de dados.

PostGreSql

O PostgreSQL é um poderoso sistema de gerenciamento de banco de dados relacional (SGBDR) de código aberto. Ele foi desenvolvido com foco em conformidade com os padrões ANSI SQL e oferece uma ampla gama de recursos avançados para atender às necessidades de aplicativos de diferentes tamanhos e complexidades.

O PostgreSQL é amplamente utilizado em diversos cenários, desde aplicativos web simples até grandes sistemas corporativos.

Modelagem de Dados (2)

1ª forma normal

1ª Forma Normal (1NF): Nesta forma, as tabelas devem ser tabulares, ou seja, cada valor em uma coluna deve ser único e indivisível, não sendo permitido agrupar múltiplos valores em uma única célula. Além disso, cada tabela deve possuir uma chave primária única, que identifica de forma exclusiva cada registro.

2ª forma normal

2ª Forma Normal (2NF): Além de atender aos critérios da 1NF, nesta forma, cada coluna que não faz parte da chave primária deve depender completamente da chave primária, e não de apenas uma parte dela. Isso significa que as dependências funcionais devem ser eliminadas, movendo as colunas dependentes para outras tabelas, evitando redundância e inconsistências nos dados.

3ª Forma Normal

3ª Forma Normal (3NF): Além de atender aos critérios da 2NF, nesta forma, não devem existir dependências transitivas entre as colunas não chave. Ou seja, uma coluna não chave não deve depender de outra coluna não chave. Para eliminar essas dependências, é necessário criar tabelas adicionais e organizar as informações de forma mais granular, evitando duplicação de dados e garantindo a integridade e a consistência das informações.

Projetos Desenvolvidos com o uso do PgAdmin4

Neste projeto nós Utilizamos o PgAdimin para criar um banco de dados, sobre uma agência

Como Construímos este projeto?

O projeto envolve a criação de um banco de dados para armazenar informações sobre departamentos, cursos, disciplinas, alunos e matrículas. Cada tabela representa uma entidade específica e possui colunas que definem os atributos dessas entidades.

Por exemplo, a tabela "Departamentos" guarda informações sobre os departamentos, como a sigla e o nome. A tabela "Cursos" está relacionada aos departamentos e guarda informações sobre os cursos, como a sigla e o nome. A tabela "Disciplinas" guarda informações sobre as disciplinas, como a sigla, o nome e se são optativas.

A tabela "Alunos" guarda informações dos alunos, como o RA, o CPF, o nome, a data de nascimento e o status. E a tabela "Disciplina_Alunos" relaciona as disciplinas com os alunos, registrando a matrícula em uma determinada disciplina.

Essas tabelas permitem o armazenamento e a consulta de informações relacionadas a departamentos, cursos, disciplinas, alunos e matrículas, facilitando a gestão desses dados.
